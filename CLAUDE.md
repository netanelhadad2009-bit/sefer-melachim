# הוראות לClaude — אתר הרב ברק (ספר מלכים)

## 🚨 חוק קריטי: לעולם אל תסיים עבודה עם קבצים לא עקובים (untracked)

### הבעיה שקרתה בעבר
בעבר נמצא שכל התמונות, המפות, וקבצי ה-PDF של יחידות הלימוד היו **קיימים רק במחשב המקומי** אבל **לא הועלו ל-git/GitHub/Vercel** — כתוצאה מכך האתר הלייב החזיר 404 על כל הקבצים האלה, והתלמידים לא הצליחו לגשת לחומרי הלימוד.

### החוק (חובה מוחלטת)
**בכל סיום משימה, לפני שאתה מודיע למשתמש שסיימת, חובה להריץ:**

```bash
git status
```

**ולבדוק:**
1. ✅ האם יש קבצים לא עקובים (`??`) שהאתר מפנה אליהם (PDF, MD, PNG, JPG, MP3, CSV)?
2. ✅ האם יש תיקיות חדשות עם תוכן שלא עוקב?
3. ✅ האם ה-HTML/JS מפנה לקבצים שלא קיימים ב-`git ls-files`?

**אם מצאת קבצים כאלה — חובה:**
- להתריע למשתמש במפורש: "⚠️ נמצאו X קבצים לא עקובים שהאתר מפנה אליהם — הם חסרים בפרודקשן ויחזירו 404"
- להציע להוסיף אותם ל-git לפני שממשיכים
- **לא לסיים** עד שהמשתמש אישר שהעניין טופל

### בדיקה הדוקה לקבצים שהאתר מפנה אליהם
לפני סיום כל משימה שעלולה להשפיע על פריסה — להריץ:

```bash
# 1. לחלץ את כל הקבצים שה-HTML מפנה אליהם
grep -oE '(href|src)="[^"]*\.(pdf|png|jpg|jpeg|gif|svg|mp3|mp4|webp|md|csv|tsv)"' index.html | sort -u

# 2. לבדוק שכל אחד מהם קיים ב-git
git ls-files | grep -E "\.(pdf|png|jpg|mp3|md|csv)$"

# 3. לבדוק קבצים ותיקיות לא עקובים
git status --short | grep "^??"
```

**אם יש פער בין (1) ל-(2) — יש בעיה. תתריע למשתמש.**

### במיוחד מסוכן
שמות קבצים בעברית או עם רווחים — קל לפספס אותם ב-`git add .` אם יש בעיה עם קידוד או אם משתמשים ב-`git add <path>` ספציפי.

**הפתרון:** תמיד להשתמש ב-`git add -A` או `git add .` מה-root של הפרויקט, או לציין במפורש כל קובץ חדש.

## 📋 תהליך סיום משימה (Definition of Done)

לפני שאומרים "סיימתי" — לעבור על הצ'קליסט:

- [ ] `git status` — אין קבצים לא עקובים שחשובים לאתר
- [ ] כל הקבצים שה-HTML מפנה אליהם קיימים ב-git
- [ ] אם נעשה `git commit` — נעשה גם `git push`
- [ ] אם יש שינויים ב-index.html — נבדק ידנית בדפדפן (או לפחות הוזכר שלא נבדק)
- [ ] אם יש שינוי שמשפיע על פרודקשן — להגיד למשתמש מתי הפריסה תסתיים ב-Vercel
